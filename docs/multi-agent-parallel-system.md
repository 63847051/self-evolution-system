# 🚀 Multi-Agent 并行处理系统 - 使用指南

**版本**: v1.0
**创建时间**: 2026-03-12
**状态**: ✅ 已部署

---

## 🎯 系统概述

这是一个**群聊 Multi-Agent 并行任务处理系统**，通过在飞书群聊中 @ 不同角色，让多个 Agent 同时处理不同任务，大幅提升效率。

---

## 👥 角色定义

### 🎯 大领导（Orchestrator）
- **触发**: `@大领导` 或 `@大领导你安排下`
- **职责**: 任务分析、智能分配、进度监督、质量把关、结果汇总
- **模型**: GLM-4.7（高推理能力）
- **成本**: 中等

### 🏠 室内设计专家
- **触发**: `@设计`
- **职责**: 所有室内设计相关任务（图纸、平面图、立面图、天花、地面、排砖、柜体、会议室）
- **模型**: GLM-4.7（中文优化）
- **成本**: 中等
- **隔离**: 只处理设计，不处理代码或日志

### 💻 技术支持专家
- **触发**: `@技术`
- **职责**: 编程、爬虫、API、数据处理、脚本开发
- **模型**: GPT-OSS-120B（免费代码专家）
- **成本**: 免费
- **隔离**: 只处理技术，不处理设计或日志

### 📋 小蓝（工作日志管理）
- **触发**: `@小蓝`
- **职责**: 工作日志记录、更新、跟踪、统计、汇总
- **模型**: GLM-4.5-Air（免费快速）
- **成本**: 免费
- **隔离**: 只处理日志，不处理设计或代码

## 🔄 使用方式

### 场景 1: 单任务分配

```
你: @大领导 设计一个客厅方案
  ↓
大领导: 检测到设计任务 → 分配给设计专家
  ↓
设计专家: 分析需求 → 生成方案
  ↓
大领导: 质量审核 → 汇总结果
```

### 场景 2: 并行多任务 ⭐ 核心能力

```
你: @大领导 今天要完成 3 个任务：
  1. @设计 修改3F更衣室排砖图
  2. @技术 写个脚本抓取数据
  3. @小蓝 更新工作日志

  ↓ （同时触发，并行执行）

设计专家: ┐
  ├─ 分析排砖方案 → 生成图纸
  └─ 耗时: 5 分钟

技术专家: ┐
  ├─ 编写爬虫脚本 → 测试
  └─ 耗时: 3 分钟

小蓝: ┐
  ├─ 读取表格 → 更新记录
  └─ 耗时: 1 分钟

  ↓ （全部完成后）

大领导: 汇总所有结果 → 质量检查 → 反馈
```

**关键优势**:
- ⚡ **并行执行** - 总耗时 = max(5, 3, 1) = 5 分钟（而非 9 分钟）
- 💰 **成本优化** - 70% 使用免费模型
- 🎯 **专业分工** - 每个专家只处理自己的职责

## 🛠️ 技术架构

### 核心脚本

1. **parallel-task-dispatcher.sh** - 并行任务分发器
   - 解析 @ 提及
   - 识别任务类型
   - 并行执行任务
   - 收集结果汇总

2. **group-chat-mention-handler.sh** - 群聊 @ 处理器
   - 监听群聊消息
   - 识别 @ 触发
   - 路由到分发器

### 配置文件

- **agents/skill-isolation-rules.md** - 角色隔离规则
- **MEMORY.md** - 角色定义和模型配置

### 日志系统

- **logs/parallel-tasks/** - 任务执行日志
- **logs/parallel-tasks/dispatch_*.log** - 分发日志
- **logs/parallel-tasks/summary_*.md** - 汇总报告

## 🔧 配置和部署

### Step 1: 配置飞书群聊

在飞书群里添加机器人，确保能接收 @ 提及。

### Step 2: 配置 OpenClaw

在 OpenClaw 配置中启用群聊消息处理：

```json
{
  "feishu": {
    "groupChat": {
      "enabled": true,
      "mentionHandler": "/root/.openclaw/workspace/scripts/group-chat-mention-handler.sh"
    }
  }
}
```

### Step 3: 测试验证

发送测试消息：
```
@大领导 测试并行任务：@设计 测试设计任务 @技术 测试技术任务
```

## 🎯 最佳实践

### 1. 明确任务边界
- ✅ 好的: `@设计 修改3F更衣室排砖图`
- ❌ 不好的: `@设计 帮我处理一下`

### 2. 合理分配任务
- 设计任务 → @设计
- 技术任务 → @技术
- 日志任务 → @小蓝

### 3. 利用并行能力
- 多个独立任务 → 一次性 @ 多个角色
- 有依赖的任务 → 明确说明顺序

### 4. 信任大领导
- 复杂任务 → 只 @大领导，让他分析分配
- 不确定任务 → 让大领导判断
