## 🔄 四步自我改进循环

### 工作流程

```
【任务完成】
    ↓
【阶段1: 自我反思】
    ├→ 评估任务分配
    ├→ 评估API选择
    ├→ 评估透明化
    ├→ 评估直接答案
    └→ 评估验证
    ↓
【阶段2: 自我批评】
    ├→ 识别问题
    ├→ 分析原因
    └→ 生成建议
    ↓
【阶段3: 提取模式】
    ├→ 提取错误模式
    ├→ 提取最佳实践
    └→ 生成可复用规则
    ↓
【阶段4: 组织记忆】
    ├→ 按主题分类
    ├→ 按严重程度分类
    ├→ 按频率排序
    └→ 生成索引
    ↓
【永久改进】
```

### 关键特点

1. **自动化** - 无需手动干预
2. **系统化** - 四步完整循环
3. **自组织** - 记忆自动分类
4. **持久化** - 改进永久保存

## 🎯 核心改进

### 1. 自我批评机制 ⭐⭐⭐⭐⭐
**v2.0**: 只有简单的评分
**v3.0**: 系统化的自我批评

**示例**:
```
阶段1: 反思
  - 我分配这个任务了吗？ [y/n]
  - 我用了最合适的API吗？ [y/n]
  - 我透明化汇报了吗？ [y/n]
  ...

阶段2: 批评
  - ⚠️ 发现问题：未选择最优API
  - 💡 改进建议：需要优化API选择策略

阶段3: 学习
  - 模式: API选择不当模式
  - 最佳实践: 根据任务类型选择API

阶段4: 组织
  - 分类到: API集成/最佳实践
  - 严重程度: medium
  - 频率: high
```

### 2. 自组织记忆 ⭐⭐⭐⭐⭐
**v2.0**: 手动管理 `.learnings/` 目录
**v3.0**: 自动组织多维度分类

**效果**:
- 快速查找: 按主题/严重程度/频率
- 智能索引: 自动更新
- 持久化: 改进永久保存

### 3. 模式提取 ⭐⭐⭐⭐⭐
**v2.0**: 无模式提取
**v3.0**: 自动提取可复用模式

**示例模式**:
- 优秀任务执行模式（5/5分）
- 一般任务执行模式（3-4分）
- 问题任务执行模式（0-2分）
